YUV格式汇总

0、前言

一张数字彩色图片在内存中的排列由图像宽(width)、高(height)、步长(stride)、像素格式(pixel format)、位宽(bit width)等因素决定。图像宽高为水平方向和垂直方向的像素数量。stride为每行起始像素的内存地址跨度，为了提高图像读取效率，stride通常会对齐到特定边界以完整读取一行像素。

像素格式在物理上有RGB、Y’CbCr(YUV)等通道格式，RGB通道是将一个像素值分解为红、绿、蓝三原色进行存储表示，YUV是将像素值分解为明度、蓝色色度和红色色度表示，在内存排列上有紧凑格式(PACKED)、平面格式(PLANAR)和半平面格式(SEMI-PLANAR)，PACKED格式下每个像素的三个通道数据在内存上相邻排布，PLANAR格式下每个通道构成一个平面，平面与平面内存上相邻，SEMI-PLANAR格式存在多个平面，部分平面按PACKED格式排布。位宽是每个像素值存储所需的比特位数，能够衡量像素值表示的动态范围，类似的表述还有位深，表示每个通道数值的动态范围，如RGB888的位深为8，位宽为24。

1、YUV444族

YUV444格式是Y、U、V三个通道的全采样格式，每个像素上都存储一组Y、U、V值，因此YUV444三个通道的采样总数为width*height*3。

1.1 PACKED

1.1.1 AYUV

AYUV是YUV444的PACKED格式简称，AYUV在YUV三个通道的基础上增加了alpha透明度通道，每个像素位宽为32位，从内存低地址到高地址分别为A、Y、U、V分量。

  • 使用指向8位无符号数的指针uint8_t* pixel读取像素值时，其像素值对应为A = pixel[0]、Y = pixel[1]、U = pixel[2]、V = pixel[3]；
  • 使用指向32位无符号数的指针uint32_t* pixel读取像素值时，需要注意字节端序，小端序下为A = *pixel & 0xff、Y = (*pixel >> 8) & 0xff、U = (*pixel >> 16) & 0xff、V = (*pixel >> 24) & 0xff；大端序下为A = (*pixel >> 24) & 0xff、Y = (*pixel >> 16) & 0xff、U = (*pixel >> 8) & 0xff、V = *pixel & 0xff；

需要注意的是，不同平台对应的AYUV物理地址顺序可能不一致：

  • Windows平台下推荐的AYUV格式为内存低地址到高地址的顺序为V、U、Y、A分量，Microsoft AYUV；
  • Linux V4L2支持的格式更多，通过Packed YUV Image Formats (8bpc)可以查看；

1.1.2 Y410

Y410是AYUV的10bit格式，用4个字节32位作为一个像素打包单元，从高位到低位分别存放A、V、Y、U四个通道，其中A分量占2个bit，V、Y、U分别占10bit。

1.1.3 Y416

Y416是AYUV的16bit格式，用8字节64位作为一个像素打包单元，A、V、Y、U四个通道的位深均为16bit。

1.2 PLANAR (I444)

I444为YUV444的PLANAR格式简称，Y、U、V三个平面按顺序在内存中依次紧密排列，平面与平面之间没有字节填充，三个平面的像素宽、高、步长等图像格式信息保持一致，唯一的区别是存储的通道不同，每个通道的位深都是8位。读取像素值时只需使用8位无符号数进行读取即可：

1.3 SEMI-PLANAR (NV24/NV42)

YUV444的SEMI-PLANAR格式的第一个平面为Y平面，第二个平面根据U、V的顺序可以分为NV24和NV42两个格式。NV24格式第二个平面为UV平面，U分量排布于V分量前；NV42格式第二个平面为VU平面，V分量在前U分量在后。Y平面和UV(VU)平面具有相同的像素高度，像素宽度上UV平面是Y平面的两倍，因此两个平面的stride大多数情况下不相同，也不一定是两倍的关系。


2、YUV422族

YUV422格式中Y通道全采样，U、V通道做1/2下采样，每两个像素共用一个U、V分量。假设Y平面每行像素采样 width 个，那么U、V像素采样 ceil(width/2) 个，在偶数宽度的情况下U、V分量的采样数是Y分量的一半，奇数宽度的情况下则向上取整。

2.1 PACKED

2.1.1 YUYV(YUY2)

8位PACKED打包的YUV422格式被称为YUYV或YUY2格式，每两个像素作为一个单元打包，Y分量与UV分量交错排列

  • Windows平台中定义了两种8bit位深的YUV422 PACKED格式，分别是按Y0、U0、Y1、V1顺序的YUY2格式和U0、Y0、V0、Y1顺序的UYVY格式；
  • Linux V4L2根据U、V的顺序定义了四种格式：U0、Y0、V0、Y1，V0、Y0、U0、Y1，Y0、U0、Y1、V0，Y0、V0、Y1、U0；
  • 像素取值时可以使用4字节作为一个宏块处理，每个宏块包含两个像素的采样值。当水平像素个数为奇数时，最后一个孤立像素无法组成宏块，一种常见的处理方法是在编码时最后一个像素单独作为一个宏块，比如像素值为[y, u, v]时，最后一个宏块为[y, u, y, v]，How does the UYVY (YUV 4:2:2, Y422, UYNV, HDYC) colour system handle odd pixel counts?

2.1.2 Y210, Y212, Y216

10bit、12bit和16bit位深的YUV422 PACKED格式称为Y210、Y212和Y216，这几个格式中每个通道分量都使用2个8位字节按小端序的”字“存储，高位是有效位，低位是0无效位。

2.2 PLANAR

2.2.1 I422

I422是YUV422的平面格式，内存上按照Y、U、V三个平面紧密排列，三个平面具有相同的高度，U、V平面的宽度为Y平面的一半，具有相同的stride。

2.3 SEMI-PLANAR

2.3.1 NV16/NV61

与NV24/NV42相同，YUV422的SEMI-PLANAR格式根据第二个平面中U、V的交错排列顺序不同可分为NV16和NV61，NV16中U分量位于V分量之前，NV61则相反。Y平面和UV平面具有相同的高度，宽度上根据奇偶像素宽度的不同：偶数个像素时Y平面与UV平面宽度相同，此时两个平面具有相同的stride；奇数个像素时UV平面比Y平面多出两个像素宽度，此时stride也不相同。


3、YUV420族

YUV420中Y分量全采样，U、V分量在垂直像素和水平像素两个方向都取1/2的半采样。对于width × height的图像，Y分量总采样数为width × height个，U、V分量在垂直方向采样 ceil(height/2) 个，水平方向采样 ceil(width/2) 个，总采样数为 ceil(width/2) × ceil(height/2) 个。

3.1 PACKED

由于YUV420的U、V分量在像素高度方向采用1/2下采样，Y、U、V三个通道交错排列的PACKED格式非常少见，目前没有一个通用的格式规范。

3.2 PLANAR

3.2.1 I420/YV12

I420/YV12格式中Y、U、V三个分量分别位于不同的平面，每个分量位深为8位，Y分量排布为 width × height，U、V分量排布为 ceil(width/2) × ceil(height/2)，I420中U平面在前V平面在后，YV12中V平面在前U平面在后。


3.2.2 IMC1/IMC3

IMC1/IMC3可以认为是YV12/I420的对齐版本，在平面步长上，U、V平面对齐到与Y平面一致，三个平面具有相同的stride，此外U、V平面的起始地址必须对齐到16行整数倍的地址上，意味着三个平面之间有填充字节。


3.2.3 IMC2/IMC4

IMC2/IMC4格式中U、V平面从平面在内存上的相邻变成了每一行分量的相邻排布，在一行U分量后紧跟着一行V分量，每行U分量与V分量行交错排布，平面不需要对齐到16行倍数地址。


3.3 SEMI-PLANAR

3.3.1 NV12/NV21

YUV420的8bit SEMI-PLANAR格式为NV12/NV21，排布类似NV16/NV61，区别在于平面高度减半。


4、YUV411族

YUV411中Y分量全采样，U、V分量在水平像素方向1/4下采样，垂直像素方向全采样。因此，Y分量采样数为 width × height，U、V分量采样数为 ceil(width/4) × height。

4.1 PACKED

4.1.1 Y41P

Y41P以3个字作为一个宏块，每个字4个字节，存储8个像素的YUV值。

4.2 PLANAR

4.2.1 YUV411P

YUV411P的三个平面具有相同的像素高度，U、V平面的像素宽度为Y平面的1/4采样。
